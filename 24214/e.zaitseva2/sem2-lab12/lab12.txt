Доказательство:

Пусть существует корректная программа.

1. Тогда Child должен быть заблокирован до первого действия Parent => существует мьютекс m1, который Child пытается lock.
Этот m1 должен быть уже захвачен кем-то (иначе Child не блокируется) => его захватил Parent

2. Теперь после выполнения Parent должен разблокировать Child => необходимо вызвать unlock(m1);
unlock(m1) должен вызвать поток, который его захватил (Parent); Теперь Child захватывает m1 и выполняется

3. Далее Child должен передать управление Parent => Parent должен быть заблокирован на каком-то мьютексе m2 =>
m2 должен быть захвачен Child => Child должен вызвать unlock(m2), чтобы разбудить Parent

Если m2 захвачен Child — unlock(m2), то тогда Parent должен был ранее попытаться lock(m2) => 
Parent блокируется на m2, захваченном Child

То есть:

Parent: lock(m2) Child: unlock(m2)
Child: lock(m1) Parent: unlock(m1)
=> deadlock => Строгое чередование потоков невозможно реализовать только с помощью двух мьютексов.
